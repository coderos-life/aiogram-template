# Политика безопасности

## Поддерживаемые версии

Шаблон следует за основной веткой `main`. Исправления безопасности должны применяться к последней версии.

## Сообщение об уязвимости

Не открывайте публичные issues для секретов, обхода авторизации, injection-векторов и других уязвимостей, которые можно использовать для атаки.

Сообщайте приватно через механизм security advisory в репозитории или напрямую владельцу репозитория.

## Базовые требования

- Никогда не коммитьте `.env`-файлы, токены ботов, пароли от базы данных или учётные данные Redis.
- Держите `detect-private-key` и другие pre-commit проверки включёнными.
- Валидируйте все внешние payloads и держите Telegram callback data компактными и детерминированными.
- Для платежей, финализации заказов и проверок прав доступа предпочитайте fail-closed поведение.
