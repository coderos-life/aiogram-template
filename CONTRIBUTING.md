# Участие в разработке

## Локальная настройка

```bash
cp .env.example .env
make install-dev
```

Перед запуском бота задайте валидный `BOT_TOKEN`. Модульные тесты могут запускаться с тестовым окружением по умолчанию из `pyproject.toml`.

## Проверки

Перед открытием запроса на слияние выполните:

```bash
make format
make check
```

`make test` пропускает PostgreSQL integration-тесты, если локальная база данных недоступна. В GitHub Actions эти тесты запускаются с PostgreSQL-сервисом.

## Миграции

Создать миграцию:

```bash
make migration message="describe change"
```

Перед коммитом проверьте сгенерированные Alembic revisions.

## Правила для запроса на слияние

- Держите изменения сфокусированными и по возможности обратно совместимыми.
- Обновляйте тесты для новых handlers, repositories, settings или поведения API.
- Не коммитьте `.env`, logs, caches и сгенерированные локальные артефакты.
- Указывайте миграции или шаги деплоя в описании PR.
